# Security Policy

## Supported Versions
Security updates are provided for the latest deployed version of this site only.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability
If you discover a potential security vulnerability, please do not open a public issue. Instead, please report it via the GitHub Security Advisory feature or email LiamTPearson@gmail.com.

This site is maintained by a single person on a best-effort basis. Once a report is received:

* It will be acknowledged as soon as reasonably possible.
* An indicative timeline for a fix will be provided once the issue has been triaged.
* Users will be notified via site updates once any patch relevant to them is live.

No formal service-level agreement is implied. Responsible disclosure is greatly appreciated.

## Scope

In scope:

* Pages hosted under `liamtp-n.github.io/Consultation-Services/`.
* Client-side code in this repository (HTML, CSS, JavaScript).
* The ROM-Based Exercise web app (`app.html`) and its Firebase integration.

Out of scope:

* Third-party services (Firebase Auth, Google Forms, Google Apps Script, Formspree, jsDelivr, Google Fonts, CDN providers). Please report issues with those directly to the respective vendor.
* Denial-of-service attacks or volumetric/load testing.
* Social-engineering or physical attacks.
